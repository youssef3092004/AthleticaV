# Mobile App API Endpoints - Complete List

## Authentication

### Client Registration with Optional Image Upload

- **POST** `/api/v1/auth/register/client` - Register new client
  - **Content-Type**: `multipart/form-data` (if uploading image) or `application/json`
  - **Form Fields**:
    - `name` (string, required) - Full name, 5-30 characters
    - `email` (string, required) - Valid email address
    - `phone` (string, required) - Phone number
    - `password` (string, required) - Minimum 8 characters
    - `profileImage` (file, optional) - Image file (jpg, png, gif, webp, max 5MB)
    - `age` (number, optional) - Age in years
    - `heightCm` (number, optional) - Height in centimeters
    - `weightKg` (number, optional) - Weight in kilograms
    - `fitnessGoal` (string, optional) - Primary fitness goal
    - `medicalConditions` (string, optional) - Any medical conditions
    - `code` (string, optional) - Trainer invite code for direct assignment
  - **Response**:
    ```json
    {
      "success": true,
      "message": "Client registered successfully",
      "data": {
        "user": {
          "id": "uuid",
          "name": "John Doe",
          "email": "john@example.com",
          "phone": "+1234567890",
          "profileImage": "/uploads/profiles/profile-1234-xyz.jpg",
          "isVerified": false,
          "createdAt": "2025-01-15T10:00:00Z"
        },
        "role": { "id": "uuid", "name": "CLIENT" },
        "activation": { "flow": "DIRECT_LOGIN", "status": "ACTIVE" }
      }
    }
    ```

### Trainer Registration with Optional Image Upload

- **POST** `/api/v1/auth/register/trainer` - Register new trainer
  - **Content-Type**: `multipart/form-data` (if uploading image) or `application/json`
  - **Form Fields**:
    - `name` (string, required) - Full name, 5-30 characters
    - `email` (string, required) - Valid email address
    - `phone` (string, required) - Phone number
    - `password` (string, required) - Minimum 8 characters
    - `profileImage` (file, optional) - Profile photo (jpg, png, gif, webp, max 5MB)
    - `yearsExperience` (number, optional) - Years of training experience
    - `rating` (number, optional) - Initial trainer rating
  - **Response**: Similar to client registration, includes trial subscription info

### Legacy Registration

- **POST** `/api/v1/auth/register` - Register new client (backwards compatible)
  - Same as `/api/v1/auth/register/client`

### Other Auth Endpoints

- **POST** `/api/v1/auth/login` - User login
- **POST** `/api/v1/auth/logout` - User logout
- **PATCH** `/api/v1/auth/resetPassword` - Reset password (requires authentication)

## Client Profile

- **GET** `/api/v1/client-profiles/:id` - Get client profile details
- **PATCH** `/api/v1/client-profiles/:id` - Update client profile

## Client Intake (Questionnaire)

- **POST** `/api/v1/client-intake/answers` - Submit/update 25-question intake answers
  - Request: `{question_key: "answer_value", ...}` (25 questions)
  - Response: `{success: true, message: "Answers updated successfully"}`
- **GET** `/api/v1/client-intake/status` - Get intake completion status

## Nutrition & Meals

- **GET** `/api/v1/nutrition/daily-summary?date=YYYY-MM-DD` - Get daily macro aggregation
  - Optional `date` parameter (defaults to today)
  - Returns: carbs, protein, fat, calories (consumed vs target with percentages)
- **GET** `/api/v1/meal-plans/:id` - Get active meal plan
- **GET** `/api/v1/meal-completions` - List completed meals
- **POST** `/api/v1/meal-completions` - Mark meal as completed
- **PATCH** `/api/v1/meal-completions/:id` - Update meal completion

## Workouts

- **GET** `/api/v1/workouts/:id` - Get workout details
- **GET** `/api/v1/workouts/:id/week-summary` - Get workout calendar for week
  - Returns: Array of 7 days with status (COMPLETED/SCHEDULED)
  - Each day: `{dayIndex, date, title, status, completedAt}`
- **GET** `/api/v1/workout-days/:id` - Get workout day details
- **PATCH** `/api/v1/workouts/:id/days/:dayId` - Update trainer note on workout day
  - Request: `{trainerNote: "string"}`
  - Response: Updated day object with trainerNote
- **POST** `/api/v1/workout-items/:id/complete` - Mark workout item as complete (legacy)
- **POST** `/api/v1/workout-completions` - Create/update workout completion
  - Request: `{workoutItemId, loggedSets, loggedReps, loggedWeightKg, completedAt, note}`
- **GET** `/api/v1/workout-completions` - List workout completions

## Streaks & Progress

- **GET** `/api/v1/streaks/current` - Get current streak data + monthly calendar
  - Returns: `{currentStreak, longestStreak, lastWorkoutAt, workoutDays: [{date, completed, dayOfWeek}]}`
- **GET** `/api/v1/progress` - Get overall progress metrics

## Chat & Messaging

- **GET** `/api/v1/conversations` - List all conversations
- **POST** `/api/v1/conversations` - Start new conversation with trainer
- **GET** `/api/v1/conversations/:id` - Get conversation details
- **GET** `/api/v1/messages?conversationId=id` - List messages in conversation
- **POST** `/api/v1/messages` - Send message
- **WebSocket** Connect to Socket.IO for real-time messages
  - Event: `new_message` - Listen for incoming messages

## Trainer Profile

- **GET** `/api/v1/trainer-profiles/:id` - Get trainer profile and availability

## Activity & Logging

- **GET** `/api/v1/activity-logs` - Get user activity history

## Implementation Notes for Mobile Dev

### Profile Image Upload (Registration)

**Client Registration with Image**:

```swift
// iOS Example - Swift
let formData = MultipartFormData()
formData.append("John Doe", forKey: "name")
formData.append("john@example.com", forKey: "email")
formData.append("+1234567890", forKey: "phone")
formData.append("password123", forKey: "password")

if let imageData = profileImage.jpegData(compressionQuality: 0.8) {
  formData.append(imageData, withFileName: "profile.jpg", mimeType: "image/jpeg", forKey: "profileImage")
}

let request = URLRequest(url: URL(string: "/api/v1/auth/register/client")!)
// Send formData...
```

**JavaScript/React Example**:

```javascript
const formData = new FormData();
formData.append("name", "John Doe");
formData.append("email", "john@example.com");
formData.append("phone", "+1234567890");
formData.append("password", "password123");

if (imageFile) {
```

**Key Points**:

- Form field name: `profileImage` (must match exactly)
- Allowed formats: JPG, PNG, GIF, WebP
- Max file size: 5MB
- Image is optional - will use default if not provided
- Returns uploaded image URL in `profileImage` field

### Daily Macro Summary

- Request: `GET /api/v1/nutrition/daily-summary`
- Query params: `date=2025-01-15` (YYYY-MM-DD format)
- Use for: Nutrition macro bars on home screen
- Returns percentages for progress visualization

### Workout Calendar

- Request: `GET /api/v1/workouts/{workoutId}/week-summary`
- Returns status dots for each day of current workout plan
- Use for: Week view calendar with completion status
- Status values: `COMPLETED`, `SCHEDULED`, `REST`, `SKIPPED`

### Trainer Notes

- When: Update trainer feedback on specific workout day
- Request: `PATCH /api/v1/workouts/{workoutId}/days/{dayId}`
- Body: `{trainerNote: "Your form was good today..."}`
- Returns: Updated day with trainerNote field

### Streaks Display

- Request: `GET /api/v1/streaks/current`
- Returns: Current streak count + longest streak + month calendar
- Calendar shows: `{date: "2025-01-15", completed: true/false}`
- Auto-updates when workouts are completed

## Rate Limiting

- API rate limit: 600 requests per 15 minutes
- After 120 requests, response times increase progressively
- Recommended: Cache responses when possible, use pagination

## Error Codes

- `400` - Bad Request (validation error)
- `401` - Unauthorized (missing/invalid token)
- `403` - Forbidden (insufficient permissions)
- `404` - Not Found
- `409` - Conflict (duplicate/state error)
- `500` - Server Error

## Client Onboarding Flow with Trainer Code

### Complete Flow: Client Joins Trainer via Invite Code

**Scenario**: New client has trainer code (e.g., `ATHLI-150`), wants to join trainer and start program.

#### Step 1: Search for Trainer by Code

```
GET /api/v1/client-invites/search/trainers?code=ATHLI-150
```

**Response**:

```json
{
  "success": true,
  "data": [
    {
      "id": "trainer-uuid",
      "name": "John Doe",
      "email": "john@trainer.com",
      "profileImage": "url",
      "trainerProfile": {
        "isVerified": true,
        "rating": 4.8,
        "yearsExperience": 5
      },
      "inviteCodeAsTrainer": {
        "code": "ATHLI-150",
        "totalClients": 25
      }
    }
  ]
}
```

#### Step 2: Client Reviews Trainer Profile

Display trainer info (name, rating, experience, current client count).

#### Step 3: Client Approves Connection

```
POST /api/v1/client-invites/approve/{clientId}
```

**Request**:

```json
{
  "trainerId": "trainer-uuid",
  "inviteCodeId": "code-uuid" // Optional if using code from search
}
```

**Response**:

```json
{
  "success": true,
  "message": "Trainer connection approved",
  "data": {
    "trainerClientId": "uuid",
    "trainerId": "trainer-uuid",
    "clientId": "client-uuid",
    "status": "ACCEPTED",
    "createdAt": "2025-01-15T10:00:00Z"
  }
}
```

This creates:

- ✅ `TrainerClient` record (links trainer to client)
- ✅ `ClientProfile` record (for client if not exists)
- ✅ Initial `ClientIntake` if configured

#### Step 4: Complete Intake Questionnaire (25 Questions)

```
POST /api/v1/client-intake/answers
```

**Request**: (25 questions answered)

```json
{
  "SLEEP_HOURS": "7-8",
  "OCCUPATION": "Software Engineer",
  "STRESS_LEVEL": "MODERATE",
  ...
  "EXPECTED_CHALLENGES": "Time management"
}
```

**Response**:

```json
{
  "success": true,
  "message": "Answers updated successfully"
}
```

#### Step 5: Trainer Creates Program

Trainer uses platform/app to create:

1. **Workout Template** → Workout Days → Workout Items (exercises)
2. **Meal Plan Template** → Meal Days → Meal Items

#### Step 6: Assign Program to Client

```
POST /api/v1/programs
```

**Request**:

```json
{
  "trainerId": "trainer-uuid",
  "clientId": "client-uuid",
  "workoutTemplateId": "template-uuid",
  "mealTemplateId": "meal-template-uuid",
  "startDate": "2025-01-16",
  "duration": 12 // weeks
}
```

**Response**:

```json
{
  "success": true,
  "data": {
    "id": "program-uuid",
    "status": "ACTIVE",
    "workoutId": "workout-uuid",
    "mealPlanId": "meal-plan-uuid"
  }
}
```

This creates:

- ✅ Active `Workout` (with WorkoutDays and WorkoutItems)
- ✅ Active `MealPlan` (with MealPlanDays and MealPlanItems)
- ✅ Client can now start logging workouts and meals

#### Step 7: Client Views Program & Starts

```
GET /api/v1/workouts/{workoutId}
```

Returns full program with:

- 📅 Week summary calendar (`GET /api/v1/workouts/{id}/week-summary`)
- 💪 Daily exercises
- 📊 Trainer notes (as coach provides feedback)

#### Step 8: Client Logs Workouts & Meals

```
POST /api/v1/workout-completions
```

```json
{
  "workoutItemId": "item-uuid",
  "loggedSets": 3,
  "loggedReps": 10,
  "loggedWeightKg": 50
}
```

✅ Auto-triggers:

- Streak update (`GET /api/v1/streaks/current`)
- Progress calculation
- Day/week completion status

#### Step 9: Track Progress

- **Daily macros**: `GET /api/v1/nutrition/daily-summary?date=2025-01-16`
- **Streaks**: `GET /api/v1/streaks/current`
- **Week calendar**: `GET /api/v1/workouts/{id}/week-summary`

#### Step 10: Chat with Trainer

```
POST /api/v1/conversations
```

Create conversation → Send messages → Trainer provides feedback

```
Program: { trainerId, clientId, status: "ACTIVE" }

```

### Key Improvements

1. **Performance**: Streak updates use `streakService` (calculates from source data)
2. **Consistency**: All calculations based on `workoutDays` + `mealPlanDays` completion status
3. **Simplification**: Response returns only `{success, message}` (no bloated data)
4. **Calendar**: Uses `DayProgress` pre-calculated data (no query per day)
