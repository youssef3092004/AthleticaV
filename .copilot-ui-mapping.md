## Screen-by-Screen Analysis

### 1. **View Profile Screen** ✅ FULLY IMPLEMENTED

**Existing Model:** `User` + `ClientProfile` + `ClientIntake`

**Displays:**

- Name, Height, Weight (from ClientProfile or ClientIntake)
- "Edit Profile" button → Routes to Edit Profile screen
- "Information" section with assigned plan + progress overview
- Progress chart (Daily/Weekly/Monthly)
- Subscription information

**Endpoint:** `GET /client-profiles/:id` (exists)
**Status:** Ready ✅

### 2. **Edit Profile Screen** ✅ FULLY IMPLEMENTED

**Model:** `ClientIntake` with 25 questions

**Displays:**

- Q1-Q25: All health, fitness, nutrition, goals questions
- Each question has options/multiselect
- Save button

**Endpoints:**

- `GET /client-intake/questions` → returns all 25 questions ✅
- `POST /client-intake/answers` → saves answers ✅
- `GET /client-intake/answers/client/:clientId` → fetches saved answers ✅

**Status:** Ready ✅

### 5. **Workouts Screen** ⚠️ NEEDS MINOR ADDITIONS

**Model:** `WorkoutDay` + `WorkoutItem` + `WorkoutCompletion`

**Displays:**

- Weekly calendar with day labels (M, T, W, Th, F, S, Su)
- Color dots indicating workout days (Completed, Scheduled, Rest)
- Day 1 through Day 7 summary tabs
- Exercises list for selected day
- Summary stats (Workouts, Hardness)
- Notes from trainer (dynamic)

**Existing Endpoints:**

- `GET /workouts/current` → gets current workout ✅
- `GET /workouts/:id/days` → gets all days ✅
- `GET /workouts/:id/days/:dayId/items` → gets exercises ✅

**MISSING - Needs to be added:**

1. **Trainer Note field** ❌
   - Add field to `WorkoutDay` model:

   ```prisma
   model WorkoutDay {
     ...
     trainerNote    String?  @db.Text  // New field
   }
   ```

   - Endpoint: `PATCH /workouts/:workoutId/days/:dayId` to update note

2. **Week Calendar Dots** ❌
   - Endpoint: `GET /workouts/:id/week-summary`
   - Returns: array of 7 days with status (COMPLETED, SCHEDULED, REST, SKIPPED)

3. **Workout Status Tracking** ⚠️
   - Need: `GET /workouts/:id/days/:dayId/status` endpoint

**Status:** 70% ready - needs 3 additions ⚠️

### 6. **Streak Calendar** ⚠️ PARTIALLY IMPLEMENTED

**Model:** `UserStreak` (exists but not used)

**Displays:**

- Monthly calendar grid with workout completion dots
- Each day shows if user worked out (purple dot) or not
- Streak count indicator

**Existing Model:**

```prisma
model UserStreak {
  id            String   @id @default(uuid())
  userId        String   @unique
  currentStreak Int      // Days in a row
  longestStreak Int
  lastWorkoutAt DateTime?
  user          User     @relation(fields: [userId], references: [id])
}
```

**MISSING - Needs:**

1. **Endpoint: GET /streaks/current**

   ```javascript
   // Returns
   {
     "currentStreak": 5,
     "longestStreak": 12,
     "workoutDays": [
       { "date": "2026-04-28", "completed": true },
       { "date": "2026-04-27", "completed": true },
       { "date": "2026-04-26", "completed": false },
       ...
     ]
   }
   ```

2. **Update Logic:**
   - When `WorkoutCompletion` is created → increment streak
   - Reset if missed a day

**Status:** 40% ready - needs endpoint + logic ⚠️
